## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding (e.g. pin tooltip px-2 py-1) |
| sm | 8px | Compact element spacing (e.g. gap-3 between FAB buttons maps to 12px — nearest sm/md boundary) |
| md | 16px | Default element spacing (e.g. map container padding left-4 = 16px) |
| lg | 24px | Bottom sheet section padding (px-6 = 24px) |
| xl | 32px | — |
| 2xl | 48px | — |
| 3xl | 64px | — |

Exceptions:
- FAB buttons: 48px (w-12 h-12) touch target — exceeds standard grid, required for mobile tap target (WCAG 2.5.5 minimum 44px)
- Zoom +/- buttons: 48px (w-12 h-12) — same FAB pattern, same exception
- FAB column gap: 12px (gap-3) — established pattern, preserved as-is
- Bottom sheet drag handle: 6px height (h-1.5) — decorative, not grid-bound
- Source: existing MapInterface.tsx and VendorBottomSheet.tsx patterns

## Interaction Contracts

### Map Image Rendering
- When `location.map_image_path` is non-null: render `<img>` as full-bleed map background, replacing the mock gradient/road divs.
- Image URL: `/maps/{id}.png?t={map_updated_at}` — append cache-busting query param.
- Image must fill the TransformComponent container with `object-fit: contain` so the full bbox is visible at 1x zoom.
- While PNG loads: show a gray skeleton (bg-gray-200 animate-pulse) the same size as the map container.
- Source: CONTEXT.md — Claude's Discretion (image loading state)

### Zoom & Pan (react-zoom-pan-pinch)
- Wrap `<img>` and pin overlay in `<TransformWrapper>` + `<TransformComponent>`.
- Bounded zoom: minScale=1, maxScale=5.
- Gestures: pinch-to-zoom on touch, mouse wheel on desktop, drag to pan.
- On-screen controls: + (ZoomIn) and - (ZoomOut) buttons in the right FAB column, positioned below the Settings and Globe FABs.
- Button appearance: w-12 h-12, bg-white, rounded-full, shadow-lg — identical to existing FAB buttons.
- Icon: `Plus` and `Minus` from lucide-react (w-5 h-5, text-gray-700).
- Source: CONTEXT.md decisions

### Vendor Pin Rendering
- Pins positioned via `style={{ top: '${y}%', left: '${x}%' }}` where x/y are projected percentages from lat/lon.
- Projection: `x = (lon - lon_nw) / (lon_se - lon_nw) * 100`, `y = (lat_nw - lat) / (lat_nw - lat_se) * 100`.
- Vendors with null lat or null lon are silently omitted — no placeholder, no count badge.
- Pin overlay div must sit inside TransformComponent so pins scale/pan with the image.
- Source: CONTEXT.md decisions + REQUIREMENTS.md INT-02

### Pin Tap Sequence (Two-Tap)
1. First tap: pin scales to 1.25x, name tooltip appears above pin (absolute -top-8, bg-gray-900, text-white, text-xs, px-2 py-1, rounded, whitespace-nowrap).
2. After 300ms auto-open OR on second tap: call `onSelectVendor(vendor)` to open VendorBottomSheet.
3. Tapping a different pin while one is selected: deselects the previous, selects new pin immediately.
4. Pin scale transition: Framer Motion scale animation, duration 200ms, ease "easeOut".
- Source: CONTEXT.md decisions (single tap → tooltip → 300ms auto-open)

### Fallback (no map image)
- When `location.map_image_path` is null: render the existing vendor list UI unchanged.
- No visual change to this path — zero regressions.
- Source: REQUIREMENTS.md MVIEW-03

### isAddingVendor Button (preserve + disable)
- + button in FAB column remains visible for admin/foodvendor roles.
- In Phase 2: button is rendered but disabled (opacity-50, cursor-not-allowed, onClick handler no-op).
- Phase 3 will re-wire this to tap-to-place flow.
- Source: CONTEXT.md decisions

## Accessibility Notes

- All FAB buttons (including + and -) must have `aria-label` attributes: "Go back", "Settings", "Language", "Zoom in", "Zoom out", "Add vendor".
- Zoom +/- buttons provide keyboard/assistive access to zoom in addition to gesture support.
- Touch targets: all interactive elements are 48px × 48px minimum (exceeds WCAG 2.5.5 44px minimum).
- Pin buttons: `type="button"` explicit, no submit behavior.
- Map image: `alt="Map of {location.name}"` (descriptive, not empty).
- Pin tooltip: shown on selection state — assistive users relying on button label (`aria-label={vendor.name}`) do not need the tooltip.
