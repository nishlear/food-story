## Spacing Scale

Declared values (must be multiples of 4). Derived from consistent patterns across
LoginScreen, SettingsModal, VendorBottomSheet, and LocationSelection.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps (`gap-1`), tight inline margins |
| sm | 8px | Compact element spacing (`p-2`, `gap-2`, `space-y-2`) |
| md | 16px | Default element spacing (`p-4`, `gap-4`, `space-y-4`) |
| lg | 24px | Section padding (`p-6`, `space-y-6`) |
| xl | 32px | Card padding (`p-8`), form sections |
| 2xl | 48px | Not used yet — reserved for major section breaks |
| 3xl | 64px | Not used yet — reserved for page-level spacing |

Exceptions: Touch targets use minimum 44px (p-3 on buttons = 12px padding + content;
full-width buttons meet this by width). No explicit 44px token needed — full-width
CTAs satisfy touch target by design.

## Color

Derived from LoginScreen.tsx, SettingsModal.tsx, LocationSelection.tsx.
Orange is the brand accent; all other color usage is neutral grays.

| Role | Tailwind Token | Hex (approx) | Usage |
|------|---------------|--------------|-------|
| Dominant (60%) | white / gray-50 | #FFFFFF / #F9FAFB | Screen backgrounds, card surfaces, modal interiors |
| Secondary (30%) | gray-100 / gray-200 | #F3F4F6 / #E5E7EB | Input backgrounds, dividers, icon container fills, bottom sheet backgrounds |
| Accent (10%) | orange-500 | #F97316 | Reserved for specific elements listed below |
| Destructive | red-500 | #EF4444 | Inline error messages only (no destructive actions in Phase 1) |

Accent (orange-500) reserved for:
- Primary CTA button background (`bg-orange-500`)
- Brand logo/icon container background
- Toggle/switch active state (`bg-orange-500`)
- Focus ring on text inputs (`focus:ring-orange-400`)
- Range slider thumb (`accent-orange-500`)
- Active icon container tint (`bg-orange-100 text-orange-600`)

Accent is NOT used for: headings, body text, borders, secondary actions, nav items,
card backgrounds, or any passive UI element.

Gradient: Login screen only uses `bg-gradient-to-br from-orange-400 to-orange-600`
as the full-screen background behind the card. This is the sole gradient in the system.

## API Response Shape Contract

Phase 2 frontend will read these fields. Declared here so executor treats them as
the visual data contract — not just a backend detail.

### Street object (GET /api/streets, GET /api/streets/:id)

```typescript
{
  id: string;
  name: string;
  city: string;
  description: string | null;
  vendors_count: number;

  // Phase 1 additions — Phase 2 reads these:
  lat_nw: number | null;
  lon_nw: number | null;
  lat_se: number | null;
  lon_se: number | null;
  map_zoom: number;           // default: 19
  map_image_path: string | null;  // e.g. "maps/1.png" — null means no map
  map_updated_at: string | null;  // ISO timestamp for cache busting
}
```

Phase 2 usage: `map_image_path !== null` is the condition for rendering MapView
instead of vendor list. Image URL: `/maps/{id}.png?v={map_updated_at}`.

### Vendor object (GET /api/streets/:id/vendors)

```typescript
{
  // existing fields unchanged...

  // Phase 1 additions — Phase 2 reads these:
  lat: number | null;   // geographic latitude
  lon: number | null;   // geographic longitude
  // NOTE: existing x/y (CSS % positions) remain untouched
}
```
